Today, many open source projects do not have the regimented patch deployment mechanisms that major players like Microsoft provide. Instead, engineers must constantly monitor each asset's respective distribution channels--email lists, issue trackers, web pages, message boards, and so on--for information about recently discovered security vulnerabilities and new patches. Of course, engineers do not do this. It is too burdensome. So they rely on luck to hear about these developments through the grapevine.

GovReady's Patchboard makes the crucial job of patch monitoring far simpler: Patchboard asks an engineer to populate a stack profile, guiding them through the process of inputting their server OS, web server, database engine(s), code libraries, and so on.

This information builds the Patchboard, a live look at the engineer's stack in terms of its security posture. GovReady maintains a dictionary of open source projects and their respective security update feeds, bringing all of this information into one place instead of forcing engineers to manually retrieve the information project-by-project. Because the Patchboard knows which versions of which packages are installed, Patchboard alerts an engineer to new issues that only apply to the stack in question.

In addition to helping engineers (both inside and outside government) work with compliant infrastructure, GovReady can provide them an easy way to monitor their stacks for new security threats and maintain their infrastructure accordingly.
